# Frontend — Исправление формы создания теста (ClassLevel-first flow)

## Контекст проблемы
Форма создания теста позволяет выбрать предмет и классы без учёта уровня класса, что приводит к:
- выбору классов разных уровней
- выбору предмета, не соответствующего уровню обучения

---

## Цель
Сделать **уровень класса (ClassLevel)** первым и обязательным шагом формы и использовать его как фильтр для:
- списка предметов
- списка классов

---

## Изменения в UI

### Последовательность выбора (обязательная)

1. Уровень класса (ClassLevel)
2. Предмет (фильтр по ClassLevel)
3. Классы (фильтр по ClassLevel)
4. Остальные настройки теста

---

## Поведение формы

### Инициализация
- Пока ClassLevel не выбран:
  - поле предмета disabled
  - поле выбора классов disabled

### При смене ClassLevel
- сбросить выбранный предмет
- сбросить выбранные классы
- заново загрузить:
  - предметы по уровню
  - классы по уровню

Запрещено сохранять старые значения.

---

## Валидация

- Нельзя отправить форму без ClassLevel
- Нельзя выбрать предмет или классы не соответствующие уровню
- Ошибки backend отображаются пользователю

---

## UX требования

- Явно обозначить шаги: **Уровень → Предмет → Классы**
- Пользователь не должен иметь возможности создать некорректный тест
- Желательно использовать stepper или секции

---

## Done Criteria

- UI не позволяет выбрать классы разных уровней
- UI не позволяет выбрать предмет без выбора уровня
- При смене уровня все зависимые поля корректно сбрасываются
